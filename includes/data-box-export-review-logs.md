3. "*コピー ログ*" でエラーを確認します。 このログには、エラーのためにコピーできなかったファイルが示されます。

    次に示すのは、エラーが発生せず、Azure から Data Box デバイスへのデータ コピー中にすべてのファイルがコピーされた場合のコピー ログのサンプル出力です。

    ```powershell
    <CopyLog Summary="Summary">
      <Status>Succeeded</Status>
      <TotalFiles_Blobs>5521</TotalFiles_Blobs>
      <FilesErrored>0</FilesErrored>
    </CopyLog>
    ``` 
    次に示すのは、コピー ログ内にエラーがあり、一部のファイルを Azure からコピーできなかった場合のサンプル出力です。


    これらのファイルをエクスポートするには、次のオプションがあります。 

    - ネットワーク経由でコピーできなかったファイルは転送できます。 
    - ご自分のデータのサイズが使用可能なデバイスの容量よりも大きい場合は、一部がコピーされ、コピーされなかったすべてのファイルがこのログに一覧表示されます。 このログを入力 XML として使用して、新しい Data Box 注文を作成し、これらのファイルをコピーすることができます。
