# Azure ゲスト OS リリースと SDK の互換性対応表
Cloud Services 向けの最新の Azure ゲスト OS リリースの最新情報を提供します。この情報は、ゲスト OS が無効になる前にアップグレード パスを計画する際に役立ちます。

> [AZURE.IMPORTANT]このページは、ゲスト OS 上で実行される Cloud Services Web ロールと worker ロールに適用されます。IaaS Virtual Machines には適用されません。ロールを構成して、「[Azure ゲスト OS の更新設定][]」に説明されているゲスト OS の自動更新を使用する場合、このページを読むことは重要ではありません。

3. 	オンプレミスのスタンドアロン VM または .NET 4.5.2 がインストールされた Azure VM で、アプリケーションの互換性の検証を行います。

## ゲスト OS のリリース情報

このセクションでは、現在サポートされているゲスト OS バージョンを示します。ゲスト OS ファミリとバージョンには、リリース日、無効になる日、有効期限があります。リリース日の時点で、ゲスト OS バージョンは、Azure クラシック ポータルで手動で選択できるようになります。ゲスト OS は、"無効になる" 日付以降に Azure クラシック ポータルから削除されます。その後は "移行中" になりますが、デプロイを更新できるように限定的にサポートされます。有効期限とは、バージョンまたはファミリが Azure システムから完全に削除されるようスケジュールされている日付です。[Azure ゲスト OS のサポートと提供終了のポリシー][retirepolicy]に関するページで詳しく説明されているように、期限切れるときにそのバージョンでまだ実行されているクラウド サービスは、停止、削除、または新しいバージョンに強制的にアップグレードされます。

マイクロソフトは、サポートされている各ゲスト OS ファミリについて少なくとも 2 つの最新バージョンをサポートしています。既存のゲスト OS バージョンが無効になる日は、デプロイできるリリース バージョンが少なくとも 2 つ残るように延期することができます。

> [AZURE.WARNING]ゲスト OS ファミリ 1 の提供終了は、2013 年 6 月 1 日に開始され、間もなく完了する予定です。このゲスト OS ファミリを使用した新規インストールの作成や古いインストールのアップグレードは行わないでください。詳細については、[Azure ゲスト OS ファミリ 1 の提供終了情報][fam1retire]に関するページを参照してください。


### ゲスト OS ファミリ、バージョン、およびリリースの説明
ゲスト OS ファミリは Microsoft Windows Server のリリース バージョンに基づいています。ゲスト OS は、Azure Cloud Services が実行される、基になるオペレーティング システムです。各ゲスト OS には、ファミリ、バージョン、リリース番号があります。

**ゲスト OS ファミリ**は、ゲスト OS の基になっている Windows Server オペレーティング システムのリリースに対応します。たとえば、ファミリ 3 は Windows Server 2012 に基づいています。

**"ゲスト OS バージョン"** は、ファミリの OS イメージに、新しいゲスト OS バージョンが作成された時点で利用可能になる関連した [Microsoft セキュリティ レスポンス センター (MSRC)][msrc] 修正プログラムを加えたものです。すべての修正プログラムが含まれているわけではありません。番号は 0 から始まり、新しい更新プログラムのセットが追加されるたびに 1 ずつ増えます。末尾のゼロが表示されるのは、重要な場合のみです。つまり、バージョン 2.10 は、バージョン 2.1 よりかなり後の異なるバージョンです。

**"ゲスト OS リリース"** は、ゲスト OS バージョンの再リリースを意味します。再リリースは、変更を必要とする問題がテスト中に見つかった場合に行われます。最新のリリースは、公開されているかどうかに関係なく、常に以前のリリースよりも優先されます。Azure クラシック ポータルでは、ユーザーは指定したバージョンの最新のリリースのみを取得できます。前のリリースで実行中のデプロイは、通常、バグの重大度に応じてアップグレードが強制されることはありません。

次の例では、2 がファミリ、12 がバージョン、"rel2" がリリースです。

**ゲスト OS リリース** - 2.12 rel2

**このリリースの構成文字列** -WA-GUEST-OS-2.12\_201208-02

ゲスト OS の構成文字列には、組み込まれているのと同じ情報に加えて、そのリリースで考慮された MSRC 修正プログラムを示す日付が含まれます。この例では、2012 年 8 月までに Windows Server 2008 R2 用に作成された MSRC 修正プログラムが含まれることが想定されます。そのバージョンの Windows Server に特に適用される修正プログラムのみが含まれます。たとえば、Microsoft Office に適用される MSRC 修正プログラムは、製品が Windows Server 基本イメージに属さないため、ここに含まれません。

## ゲスト OS システムの更新プロセス
このページには、今後予定されているゲスト OS のリリースに関する情報が含まれます。"自動" 更新に設定している場合はクラウド サービスのロールが再起動されるため、リリースのタイミングを把握する必要があるとお客様から指摘を受けています。ゲスト OS のリリースは、通常、毎月第 2 火曜日に発生する MSRC 更新プログラムのリリースから少なくとも 5 日以内に発生します。新しいリリースには、ゲスト OS ファミリごとに関連するすべての MSRC 修正プログラムが含まれます。

Microsoft Azure では、常に更新プログラムがリリースされています。ゲスト OS は、パイプラインにおいて唯一のこのような更新です。リリースは、ここに記載できないほどの多く要因の影響を受ける可能性があります。さらに、Azure は、実際には数十万台のコンピューターで実行されています。つまり、ロールが再起動される正確な日時を指定することはできません。[ゲスト OS の更新 RSS フィード][rss]は最新の情報で更新されていますが、その時刻はおおよその時間帯であると考えてください。これは問題視されているため、再起動の制限や時間の指定をできるように取り組んでいます。

ゲスト OS の新しいリリースが公開されると、Azure 全体で完全に反映されるまでに時間がかかります。サービスが新しいゲスト OS に更新されると、更新ドメインに従って再起動されます。"自動" 更新プログラムを使用するように設定されているサービスで、まずリリースが取得されます。更新後、Azure クラシック ポータルでは、ご使用のサービスの新しいゲスト OS バージョンの一覧が表示されます。この期間に再リリースが発生する場合があります。一部のバージョンは長期間にわたってデプロイされることがあります。また、正式なリリース日以降何週間も自動アップグレードの再起動が行われないこともあります。ゲスト OS が使用可能になると、ポータルや構成ファイルから明示的にそのバージョンを選択できます。

再起動の関する有益な情報と、ゲスト OS とホスト OS の更新プログラムの技術的な詳細のヒントについては、MSDN の[OS のアップグレードによるロール インスタンスの再起動][restarts]に関するブログ記事を参照してください。

ゲスト OS を手動で更新する場合は、[ゲスト OS の提供終了のポリシー][retirepolicy]に関するページを参照してください。

## ニュース アーカイブ

###### **2014 年 11 月 11 日**

11 月のリリース (4.14、3.21、2.33) が 11 月 11 日にロールアウトされました。この更新プログラムは、MSRC 更新プログラム ([Microsoft セキュリティ情報 MS14-066 - 緊急][MS14-066]) に含まれるため、以前推進されました。自動更新の Web ロールと worker ロールは、数日以内に 1 回再起動され、この修正プログラムを受信する必要があります。

###### **2014 年 11 月 10 日**
10 月のリリース (4.13、3.20、2.32) が無効になる日は、お客様からのフィードバックに基づいて更新されました。無効になる日は必ず、リリース日から少なくとも 2 か月後になります。

###### **2014 年 11 月 4 日**
10 月のリリース (4.13、3.20、2.32) は、2014 年 11 月 4 日にロールアウトされました。これには、8 月と 9 月のリリースで問題の原因となった MSRC 修正プログラムが含まれます。この問題を回避するために、10 月のリリースにはプレインストールされた .NET 3.5 および 3.5.1 が含まれていますが、無効になっています。.NET 3.5 または 3.5.1 のインストールを試みるスクリプトは、実際にこれをもう一度有効にして、.NET インストールの "成功" を返します。ただし、MSRC 修正プログラムによって作成されたフル インストールの問題も回避します。

**2014 年 10 月 20 日。更新日: 2014 年 11 月 4 日** - 9 月のリリース (4.12、3.19、2.31、1.39) は、ファミリ 3 または 4 での .NET 3.5 または 3.5.1 のインストールが失敗する原因となる [MSRC 修正プログラム MS14-046][MS14-046] により、部分的にロールアウトされました。.NET 3.5.x は、いずれのファミリでも正式にサポートされていませんが、一部のお客様がインストールにこれを使用しており、変更が発表されていないため、Microsoft は動作の変更に応じています。前のゲスト OS が無効になる日 (6 月と 7 月) は、完全にリリースされたゲスト OS が少なくとも 2 つサポートされて使用できるように、それぞれ延期されます。.NET インストールの問題の解決策は、2014 年 10 月のリリースで発表されました。

この 10 月のリリースには、プレインストールされている (ただし無効になっている) .NET 3.5 および 3.5.1 が含まれています。さらに、前に記載されている MSRC 修正プログラムも含まれています。.NET 3.5 または 3.5.1 のインストールを試みるスクリプトは、実際にこれをもう一度有効にして、.NET インストールの "成功" を返します。ただし、MSRC 修正プログラムによって作成されたインストールの問題を回避もします。

最新の 2 つのリリースが部分的にロールアウトされたため、自動更新のユーザーや新規インストールをロールアウトしたユーザーは、これらのゲスト OS のリリースのいずれかを使用している可能性があります。次の表では、ファミリ 3 と 4 での .NET 3.5 または 3.5.1 のインストールを許可するゲスト OS のリリースを示しています。現時点では、リリースでインストールが許可されている場合、MSRC 修正プログラム MS14-046 がインストールされていないことを意味します。
