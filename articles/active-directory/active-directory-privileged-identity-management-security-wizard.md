# Azure Privileged Identity Management セキュリティ ウィザード

Azure Privileged Identity Management を初めて実行すると、ウィザードが表示されます。このウィザードでは、特権 ID のセキュリティ リスクと、Privileged Identity Management を使用してリスクを軽減する方法がわかります。

確認するセクションは、**[管理者によるリスクの可能性]、[管理者の攻撃対象領域を管理する]**、**[一時的な管理者の設定を定義する]** の 3 つです。各セクションでは、概念の概要と、実行する操作の説明が示されています。

初期状態では、グローバル管理者はすべて永続的です。**[管理者によるリスクの可能性]** をクリックすると、グローバル管理者ロールの一覧と、それぞれの現在の人数が表示されます。

**[管理者の攻撃対象領域を管理する]** をクリックすると、一時的な管理者に変更したり、永続的管理者のままにしたり、ロールから完全に削除したりできます。

**[一時的な管理者の設定を定義する]** では、一時的管理者が特権を持っている期間を決定したり、通知を有効にしたり、多要素認証を要求したりできます。

## グローバル管理者ロールを一時的または永続的に変更する

グローバル管理者の期間の変更に関しては 3 つのオプションがあります。

1.  すべてのグローバル管理者を一時的にするには、**[すべてを一時的にする]** ボタンをクリックします。

2.  すべてのグローバル管理者を永続的にするには、**[すべてを永続的にする]** ボタンをクリックします。

3.  個々のグローバル管理者について変更するには、**[永続的を維持]**、**[一時的に変更]**、**[ロールから削除]** を選択します。

4.  **[OK]** をクリックします。

5.  **[Submit]** をクリックします。

## グローバル管理者ロールのアクティブ化期間を変更します。

1.  **[アクティブ化期間]** スライダーを左右に移動して、アクティブ化の期間を延長または短縮します。アクティブ化期間は最長 72 時間まで指定できます。

2.  スライダーの右側にある **[時間]** フィールドに時間数を入力します。

## 通知を有効にする

ロールがアクティブになったときに管理者がメールを受け取れるよう、**[有効]** ボタンをクリックして通知を有効にします。この機能を無効にすることもできます。

## 多要素認証を要求する

管理者がアカウントにログインしてロールの延長を要求するときに MFA を使用する必要があるようにする場合は、**[有効]** ボタンをクリックして MFA を有効にします。この機能を無効にすることもできます。

これらの設定を適用するロールを選択します。**[OK]** をクリックします。

> [AZURE.WARNING]一時的なセキュリティ管理者のロール割り当ての有効期限が切れ、そのユーザーに MFA が設定されていない場合、ユーザーは PIM をまったく管理できなくなるの、この時点で、セキュリティ管理者を複数にしておくことが重要です。

終了したら、**[OK]** ボタンをクリックします。

変更を行った後は、ウィザードは表示されなくなります。ただし、[ユーザーの管理] の下の [ウィザード] ボタンをクリックして再びアクセスできます。
